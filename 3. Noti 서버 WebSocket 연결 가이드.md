# Notifier-App WebSocket 연결 가이드

Waldlust Platform의 **notifier-api**는 애이전트에게 주문 관련 실시간 알림 전송을 위해 WebSocket 연결을 제공합니다. 본 문서는 클라이언트에서 WebSocket에 안전하게 연결하고 알림을 수신하는 방법을 안내합니다.

## 🔧 사전 준비

WebSocket 연결을 위해서는 다음 과정이 필요합니다.

### **1. Project API Key 발급**

**발트루스트(Waldlust)에서 제공하는 정보:**
- **Project ID**: 프로젝트를 식별하는 고유 값입니다. (브랜드별로 1개의 프로젝트가 할당됩니다. 예: 매머드, 텐퍼센트 등 각 브랜드마다 별도의 Project ID가 부여됩니다.)
- **Project API Key**: 원본 인증 키 (평문)
- **Shop Code**: 매장을 구분하는 고유 코드입니다. (구 PID)
  - 예시: `TEST00001`  
  - 앞의 4글자(`TEST`)는 프로젝트(브랜드)별로 정해진 약어입니다.  
  - 뒤의 5자리 숫자(`00001`)는 매장이 생성되는 순서대로 부여됩니다.
  즉, "프로젝트 약어 4글자 + 매장번호 5자리"로 구성됩니다.
  - shopCode는 에이전트 로그인시 사용한 아이디값과 동일합니다.

### **2. Project API Key 암호화**

보안을 위해 Project API Key는 발트루스트에서 제공한 암호화 방식으로 암호화해야 합니다.

**암호화 과정:**
```javascript
// 예제: 발트루스트 제공 암호화 함수 사용
const projectApiKey = "your-plain-api-key";
const encryptedApiKey = encryptApiKey(projectApiKey); // 발트루스트 제공 함수

// Authorization 헤더에 사용
const authHeader = `Bearer ${encryptedApiKey}`;
```

**주의사항:**
- **원본 Project API Key를 직접 전송하지 마세요**
- **반드시 발트루스트 제공 암호화 방식을 사용하세요**
- **암호화된 키만 Authorization 헤더에 포함하세요**

### **3. 연결 정보 확인**

연결 전 다음 정보가 준비되었는지 확인하세요:

| 항목 | 설명 | 예시 | 상태 |
|------|------|------|------|
| Project ID | 발트루스트 제공 프로젝트 ID | `project-123` | ☐ 준비완료 |
| 원본 API Key | 발트루스트 제공 원본 키 | `plain-api-key-123` | ☐ 준비완료 |
| 암호화된 API Key | 암호화 처리된 키 | `encrypted-key-abc` | ☐ 준비완료 |
| Shop Code | 매장 식별 코드 | `shop-abc` | ☐ 준비완료 |
| 암호화 함수 | 발트루스트 제공 암호화 방식 | `가이드 문서` | ☐ 준비완료 |

## 📡 연결 개요

### **기본 정보**
- **프로토콜**: WebSocket (WSS 권장)
- **엔드포인트**: `/ws`
- **인증 방식**: Bearer key + 커스텀 헤더 (key = 발트루스트에서 제공한 암호화 방식으로 암호화한 값)
- **메시지 형식**: JSON
- **연결당 매장 수**: 1개 (1:1 매핑)
- **기본 포트**: 8082 (로컬 개발시)

### **서버 환경별 URL**

| 환경 | WebSocket URL | 설명 |
|------|---------------|------|
| **Live** | `wss://notifier-api.waldplatform.com/ws` | 운영 서버 |
| **Staging** | `wss://notifier-stgapi.waldplatform.com/ws` | 테스트 서버 |

### **아키텍처**
```
[클라이언트] ←→ [WebSocket Handler] ←→ [Session Manager] ←→ [Redis Event Listener]
                     ↓
              [Project API Key 검증]
```

## 🔐 인증 및 헤더

WebSocket 연결시 다음 헤더들이 **필수**입니다:

### **필수 헤더**

| 헤더명 | 설명 | 예시 | 필수 여부 |
|--------|------|------|----------|
| `Authorization` | Bearer 토큰 인증 | `Bearer your-api-key-here` | ✅ 필수 |
| `X-Waldlust-ProjectId` | 프로젝트 식별자 | `project-123` | ✅ 필수 |
| `X-Waldlust-ShopCode` | 매장 코드 | `shop-abc` | ✅ 필수 |

### **헤더 검증 과정**

1. **Authorization 헤더 확인**
   - 발트루스트에서 공유한 암호화 방식으로 암호화를 진행했는지 복호화 과정을 통해 확인
   - `Bearer ` , 접두사 포함 여부
   - API 키 값이 비어있지 않은지 확인

2. **Project ID 검증**
   - 헤더 존재 여부
   - 유효한 프로젝트인지 확인

3. **Shop Code 검증**
   - 헤더 존재 여부
   - 매장 코드 형식 확인

4. **API 키 인증**
   - 프로젝트와 API 키의 매칭 확인
   - 권한 및 활성화 상태 검증

## ⚠️ 에러 처리

### **주요 에러 코드**

| 에러 코드 | 설명 | 해결 방법 |
|-----------|------|----------|
| `UNAUTHORIZED` | 인증 실패 | API 키 및 프로젝트 ID 확인 |
| `SESSION_NOT_FOUND` | 세션을 찾을 수 없음 | 재연결 시도 |
| `CONNECTION_FAILED` | 연결 실패 | 네트워크 상태 확인 |
| `MESSAGE_SEND_FAILED` | 메시지 전송 실패 | 연결 상태 확인 후 재시도 |
| `SESSION_CLOSED` | 세션 종료됨 | 새로운 연결 설정 |
| `REDIS_CONNECTION_FAILED` | Redis 연결 실패 | 서버 상태 확인 |
| `INVALID_EVENT_FORMAT` | 잘못된 이벤트 형식 | 이벤트 발행 로직 확인 |

### **에러 처리 예제**

```javascript
socket.onmessage = function(event) {
    const data = JSON.parse(event.data);
    
    if (data.type === 'error') {
        console.error('WebSocket 에러:', data.error);
        
        switch (data.error.code) {
            case 'UNAUTHORIZED':
                // 인증 정보 갱신 후 재연결
                reconnectWithNewAuth();
                break;
            case 'SESSION_CLOSED':
                // 즉시 재연결
                setTimeout(reconnect, 1000);
                break;
            case 'CONNECTION_FAILED':
                setTimeout(reconnect, Math.pow(2, retryCount) * 1000);
                break;
            default:
                console.error('알 수 없는 에러:', data.error);
        }
    } else {
        // 정상 이벤트 처리
        handleEvent(data);
    }
};
```

```javascript
class WaldlustNotificationClient {
    constructor(config) {
        this.config = config;
        this.socket = null;
        this.retryCount = 0;
        this.maxRetries = 5;
        this.reconnectDelay = 1000;
    }

    connect() {
        const wsUrl = `${this.config.baseUrl}/ws`;
        
        // WebSocket 연결 (브라우저는 헤더 설정이 제한적)
        this.socket = new WebSocket(wsUrl);
        
        this.socket.onopen = this.onOpen.bind(this);
        this.socket.onmessage = this.onMessage.bind(this);
        this.socket.onerror = this.onError.bind(this);
        this.socket.onclose = this.onClose.bind(this);
    }

    onOpen(event) {
        console.log('✅ WebSocket 연결 성공');
        this.retryCount = 0;
        
        // 인증 메시지 전송 (헤더 대신)
        this.authenticate();
    }

    authenticate() {
        const authMessage = {
            type: 'authenticate',
            projectId: this.config.projectId,
            shopCode: this.config.shopCode,
            apiKey: this.config.apiKey
        };
        
        this.socket.send(JSON.stringify(authMessage));
    }

    onMessage(event) {
        try {
            const data = JSON.parse(event.data);
            
            if (data.type === 'error') {
                this.handleError(data.error);
            } else if (data.type === 'auth_success') {
                console.log('🔐 인증 성공');
            } else {
                this.handleEvent(data);
            }
        } catch (error) {
            console.error('메시지 파싱 실패:', error);
        }
    }

    handleEvent(event) {
        console.log('📨 이벤트 수신:', event);
        
        // 이벤트 타입별 처리
        switch (event.eventType) {
            case 'ORDER_CREATED':
                this.onOrderCreated(event.payload);
                break;
            case 'PAYMENT_COMPLETED':
                this.onPaymentCompleted(event.payload);
                break;
            default:
                console.log('알 수 없는 이벤트 타입:', event.eventType);
        }
    }

    handleError(error) {
        console.error('🚨 WebSocket 에러:', error);
        
        // 에러 타입별 처리
        if (error.code === 'UNAUTHORIZED') {
            // 인증 갱신 로직
            this.refreshAuthAndReconnect();
        }
    }

    onError(event) {
        console.error('WebSocket 연결 오류:', event);
    }

    onClose(event) {
        console.log('WebSocket 연결 종료:', event.code, event.reason);
        
        if (this.retryCount < this.maxRetries) {
            this.retryCount++;
            const delay = this.reconnectDelay * Math.pow(2, this.retryCount - 1);
            console.log(`${delay}ms 후 재연결 시도 (${this.retryCount}/${this.maxRetries})`);
            
            setTimeout(() => this.connect(), delay);
        } else {
            console.error('최대 재연결 시도 횟수 초과');
        }
    }

    disconnect() {
        if (this.socket) {
            this.socket.close();
            this.socket = null;
        }
    }

    // 이벤트 핸들러들
    onOrderCreated(payload) {
        console.log('새 주문:', payload);
        // UI 업데이트 로직
        this.showNotification(`새 주문이 들어왔습니다: ${payload.customerName}`);
    }

    onPaymentCompleted(payload) {
        console.log('결제 완료:', payload);
        // UI 업데이트 로직
        this.showNotification(`결제가 완료되었습니다: ${payload.amount}원`);
    }

    showNotification(message) {
        // 브라우저 알림 또는 UI 업데이트
        if (Notification.permission === 'granted') {
            new Notification('Waldlust 알림', { body: message });
        }
    }
}

// 사용 예제

    baseUrl: 'wss://api.waldlust.com',
    projectId: 'project-123',
    shopCode: 'shop-abc',
    apiKey: 'your-api-key-here'
});

client.connect();
```

```javascript
class WaldlustNotificationClient {
    constructor(config) {
        this.config = config;
        this.socket = null;
        this.retryCount = 0;
        this.maxRetries = 5;
    }

    connect() {
        const wsUrl = `${this.config.baseUrl}/ws`;
```

### **Python**

```python
import asyncio
import websockets
import json
import logging

class WaldlustNotificationClient:
    def __init__(self, config):
        self.config = config
        self.websocket = None
        self.retry_count = 0
        self.max_retries = 5
        
    async def connect(self):
        """WebSocket 연결"""
        url = f"{self.config['base_url']}/ws"
        headers = {
            'Authorization': f"Bearer {self.config['api_key']}",
            'X-Waldlust-ProjectId': self.config['project_id'],
            'X-Waldlust-ShopCode': self.config['shop_code']
        }
        
        try:
            self.websocket = await websockets.connect(url, extra_headers=headers)
            logging.info("✅ WebSocket 연결 성공")
            self.retry_count = 0
            
            # 메시지 수신 루프 시작
            await self.listen()
            
        except Exception as e:
            logging.error(f"연결 실패: {e}")
            await self.retry_connect()
    
    async def listen(self):
        """메시지 수신 루프"""
        try:
            async for message in self.websocket:
                data = json.loads(message)
                
                if data.get('type') == 'error':
                    await self.handle_error(data['error'])
                else:
                    await self.handle_event(data)
                    
        except websockets.exceptions.ConnectionClosed:
            logging.warning("WebSocket 연결이 종료되었습니다")
            await self.retry_connect()
        except Exception as e:
            logging.error(f"메시지 수신 오류: {e}")
            await self.retry_connect()
    
    async def handle_event(self, event):
        """이벤트 처리"""
        logging.info(f"📨 이벤트 수신: {event}")
        
        event_type = event.get('eventType')
        payload = event.get('payload', {})
        
        if event_type == 'ORDER_CREATED':
            await self.on_order_created(payload)
        elif event_type == 'PAYMENT_COMPLETED':
            await self.on_payment_completed(payload)
    
    async def handle_error(self, error):
        """에러 처리"""
        logging.error(f"🚨 WebSocket 에러: {error}")
        
        if error.get('code') == 'UNAUTHORIZED':
            # 인증 갱신 로직
            pass
    
    async def retry_connect(self):
        """재연결 시도"""
        if self.retry_count < self.max_retries:
            self.retry_count += 1
            delay = 2 ** (self.retry_count - 1)
            
            logging.info(f"{delay}초 후 재연결 시도 ({self.retry_count}/{self.max_retries})")
            await asyncio.sleep(delay)
            await self.connect()
        else:
            logging.error("최대 재연결 시도 횟수 초과")
    
    async def on_order_created(self, payload):
        """주문 생성 이벤트 처리"""
        print(f"새 주문: {payload}")
    
    async def on_payment_completed(self, payload):
        """결제 완료 이벤트 처리"""
        print(f"결제 완료: {payload}")

# 사용 예제
async def main():
    client = WaldlustNotificationClient({
        'base_url': 'wss://notifier-api.waldplatform.com',  # Live 서버
        # 'base_url': 'wss://notifier-stgapi.waldplatform.com',  # Test 서버
        # 'base_url': 'ws://localhost:8082',  # Local 개발환경
        'project_id': 'project-123',
        'shop_code': 'shop-abc',
        'api_key': 'your-api-key-here'
    })
    
    await client.connect()

if __name__ == "__main__":
    asyncio.run(main())
```
