# API 시그니처 생성 가이드

## 개요

이 문서는 Waldlust Platform API의 보안 인증을 위한 HMAC 시그니처 생성 방법을 설명합니다.  
클라이언트에서 AUTH 관련 API 요청 시 반드시 올바른 시그니처를 생성하여 전송해야 합니다.

## 사전 준비

**발트루스트(Waldlust)에서 제공하는 정보:**
- **Project ID**: 프로젝트를 식별하는 고유 값입니다. (브랜드별로 1개의 프로젝트가 할당됩니다. 예: 매머드, 텐퍼센트 등 각 브랜드마다 별도의 Project ID가 부여됩니다.)
- **API Key**: 원본 인증 키 (평문)
- **Shop Code**: 매장을 구분하는 고유 코드입니다. (구 PID)
  - 예시: `TEST00001`  
  - 앞의 4글자(`TEST`)는 프로젝트(브랜드)별로 정해진 약어입니다.  
  - 뒤의 5자리 숫자(`00001`)는 매장이 생성되는 순서대로 부여됩니다.
  즉, "프로젝트 약어 4글자 + 매장번호 5자리"로 구성됩니다.
  - shopCode는 에이전트 로그인시 사용한 아이디값과 동일합니다.

## 시그니처 사용 API 목록
- /v0/auth/api-key/validate (project api key 버전 확인 및 최신화 api)
- /v0/auth/token/issue (POS, KIOSK 등 외부 시스템에서 발트 core api를 이용하기 위한 토큰 발급 api)

## 시그니처 알고리즘

- **해시 알고리즘**: HMAC-SHA512
- **인코딩**: Base64
- **문자 인코딩**: UTF-8

## 시그니처 생성 과정

### 1. 페이로드 데이터 준비

API 키 검증을 위한 페이로드는 다음 구조를 가집니다:

```json
{
  "projectId": "프로젝트 ID",
  "version": 1,
  "requestSource": "요청 출처",
  "datetime": "요청 시간(UTC)"
}
```

#### 필드 설명

| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `projectId` | String | 프로젝트 고유 식별자 | `"0mm8487sf4zjt"` |
| `version` | Integer | API 버전 | `1` |
| `requestSource` | String | 요청 출처 | `"POS"`, `"APP"`, `"KIOSK"` 등 |
| `datetime` | String | ISO-8601 형식의 요청 시간 (UTC)| `"2025-08-20T15:36:05.338"` |

#### RequestSource 열거형

```
APP, AGENT, ADMIN, POS, KIOSK, EXTERNAL_SYSTEM
```

### 2. JSON 직렬화

페이로드 객체를 JSON 문자열로 변환합니다.

**중요사항:**
- 공백 없는 compact 형식 사용
- 키 순서는 삽입 순서 유지
- datetime은 ISO-8601 형식 (시간대 정보 없음)

**올바른 JSON 예시:**
```json
{"projectId":"0mm8487sf4zjt","version":1,"requestSource":"POS","datetime":"2025-08-20T15:36:05.338"}
```

### 3. HMAC-SHA512 계산

1. Project API Key를 UTF-8로 인코딩하여 비밀 키로 사용
2. JSON 페이로드를 UTF-8로 인코딩
3. HMAC-SHA512 알고리즘으로 해시 계산
4. 결과를 Base64로 인코딩

### 4. 시간 검증

- 서버는 요청 시간과 현재 시간의 차이를 검증합니다.
- 기본 허용 오차: **5분**
- 시간대는 UTC 기준으로 계산됩니다.

## 구현 예시

### Python

```python
import hmac
import hashlib
import base64
import json

def compute_hash(secret: str, payload: str) -> str:
    """HMAC-SHA512 시그니처 생성"""
    secret_bytes = secret.encode('utf-8')
    payload_bytes = payload.encode('utf-8')
    
    mac = hmac.new(secret_bytes, payload_bytes, hashlib.sha512)
    return base64.b64encode(mac.digest()).decode('utf-8')

def create_json_payload(data: dict) -> str:
    """JSON 페이로드 생성"""
    def json_default(obj):
        if isinstance(obj, datetime):
            return obj.strftime('%Y-%m-%dT%H:%M:%S.%f')[:-3]
        raise TypeError(f"Object of type {type(obj)} is not JSON serializable")
    
    return json.dumps(
        data,
        separators=(',', ':'),
        ensure_ascii=False,
        default=json_default,
        sort_keys=False
    )
```

### JavaScript

```javascript
const crypto = require('crypto');

function computeHash(secret, payload) {
    const hmac = crypto.createHmac('sha512', secret);
    hmac.update(payload, 'utf8');
    return hmac.digest('base64');
}

function createJsonPayload(data) {
    return JSON.stringify(data, (key, value) => {
        if (value instanceof Date) {
            return value.toISOString().slice(0, -1); // 'Z' 제거
        }
        return value;
    });
}
```

## API 요청 예시

### 요청 헤더
```
POST /v0/auth/api-key/validate
Content-Type: application/json
Waldlust-Signature: generated_signature_here
```

### 요청 본문
```json
{
  "projectId": "0mm8487sf4zjt",
  "version": 1,
  "requestSource": "POS",
  "datetime": "2025-08-20T15:36:05.338"
}
```

**중요**: 시그니처는 `Waldlust-Signature` 헤더로 전달되며, 요청 본문에는 포함되지 않습니다.

### HTTP 클라이언트 구현 예시

#### Python (requests)
```python
import requests

# 시그니처 생성 후
headers = {
    'Content-Type': 'application/json',
    'Waldlust-Signature': signature
}

request_body = {
    "projectId": "0mm8487sf4zjt",
    "version": 1,
    "requestSource": "POS",
    "datetime": "2025-08-20T15:36:05.338"
}

response = requests.post(
    'https://api.waldlust.com/v0/auth/api-key/validate',
    headers=headers,
    json=request_body
)
```

#### JavaScript (fetch)
```javascript
const headers = {
    'Content-Type': 'application/json',
    'Waldlust-Signature': signature
};

const requestBody = {
    projectId: "0mm8487sf4zjt",
    version: 1,
    requestSource: "POS",
    datetime: "2025-08-20T15:36:05.338"
};

fetch('https://api.waldlust.com/v0/auth/api-key/validate', {
    method: 'POST',
    headers: headers,
    body: JSON.stringify(requestBody)
});
```

#### cURL
```bash
curl -X POST https://api.waldlust.com/v0/auth/api-key/validate \
  -H "Content-Type: application/json" \
  -H "Waldlust-Signature: generated_signature_here" \
  -d '{
    "projectId": "0mm8487sf4zjt",
    "version": 1,
    "requestSource": "POS",
    "datetime": "2025-08-20T15:36:05.338"
  }'
```

## 보안 고려사항

### 1. API 키 보안
- Project API Key는 절대 클라이언트 코드에 하드코딩하지 마세요
- 환경 변수나 안전한 설정 파일에 저장하세요
- 정기적으로 API 키를 갱신하세요

### 2. 시간 동기화
- 클라이언트와 서버의 시간이 동기화되어야 합니다
- NTP 서버를 사용하여 시간을 정확히 맞추세요

### 3. 네트워크 보안
- HTTPS를 사용하여 통신을 암호화하세요
- 시그니처가 노출되더라도 시간 제한으로 인해 재사용이 제한됩니다

## 문제 해결

### 일반적인 오류

#### 1. 시그니처 불일치
**증상**: `Invalid signature` 오류
**원인**: 
- 잘못된 JSON 직렬화 (공백, 키 순서)
- 다른 해시 알고리즘 사용
- 인코딩 문제

**해결방법**:
- JSON 형식을 정확히 맞추세요
- HMAC-SHA512를 사용하는지 확인하세요
- UTF-8 인코딩을 사용하세요

#### 2. 시간 오류
**증상**: `Request time out of tolerance` 오류
**원인**: 클라이언트와 서버의 시간 차이가 5분을 초과

**해결방법**:
- 시스템 시간을 NTP로 동기화하세요
- 시간대 설정을 확인하세요

#### 3. 잘못된 요청 출처
**증상**: `Invalid request source` 오류
**원인**: 지원하지 않는 RequestSource 값

**해결방법**:
- 정의된 RequestSource 열거형 값을 사용하세요
