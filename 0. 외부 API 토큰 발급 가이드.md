## 개요

tf-platform의 External API는 JWT(JSON Web Token) 기반의 인증 시스템을 사용합니다. 모든 API 요청은 유효한 액세스 토큰을 필요로 하며, 이 문서는 토큰을 발급받고 사용하는 방법을 안내합니다.

### 토큰 기반 인증의 장점
- **Stateless**: 서버가 세션을 유지할 필요가 없어 확장성이 뛰어남
- **보안성**: RSA-2048 암호화를 통한 강력한 보안
- **표준화**: JWT는 산업 표준으로 다양한 라이브러리 지원
- **만료 관리**: 토큰 만료 시간(7일)을 통한 보안 강화

### 지원하는 인증 방법

tf-platform은 두 가지 인증 방법을 지원합니다:

| 방법 | 사용 사례 | 대상 시스템 |
|------|----------|------------|
| **Project API Key 인증** | 시스템 간 자동화된 통신 | POS, KIOSK, APP, AGENT, EXTERNAL_SYSTEM |
| **매장 관리자 로그인** | 사용자 계정 기반 인증 | 매장 관리자 계정을 가진 애플리케이션 |

## 인증 방법

### 방법 1: Project API Key 인증

Project API Key 인증은 시스템 간 통신에 사용되는 가장 일반적인 인증 방법입니다. HMAC-SHA512 서명을 사용하여 API Key의 유효성을 검증하고, JWT 토큰을 발급받습니다.

#### 인증 흐름

```
┌─────────────┐                          ┌──────────────┐
│   Client    │                          │  API Server  │
└──────┬──────┘                          └──────┬───────┘
       │                                        │
       │  1. Signature 생성                    │
       │     (HMAC-SHA512)                     │
       │                                        │
       │  2. POST /v0/auth/token/issue         │
       ├───────────────────────────────────────>│
       │     Header: Waldlust-Signature         │
       │     Body: {projectId, shopCode, ...}   │
       │                                        │
       │                          3. Signature 검증
       │                             & 토큰 생성│
       │                                        │
       │  4. 응답: JWT 토큰 (7일 유효)        │
       │<───────────────────────────────────────┤
       │                                        │
       │  5. API 요청                          │
       ├───────────────────────────────────────>│
       │     Header: Authorization: Bearer {token}
       │                                        │
```

#### 1.1 토큰 발급 (Token Issue)

##### API 엔드포인트
```
POST /v0/auth/token/issue
```

##### 요청 헤더
```http
Content-Type: application/json
Waldlust-Signature: {HMAC-SHA512 서명}
```

##### 요청 본문
```json
{
  "projectId": "프로젝트 ID",
  "shopCode": "매장 코드 (예: WALD00001)",
  "requestSource": "POS | KIOSK | APP | AGENT | EXTERNAL_SYSTEM",
  "datetime": "2025-12-10T10:30:00"
}
```

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `projectId` | String | 필수 | 프로젝트 식별자 |
| `shopCode` | String | 필수 | 매장 코드 (예: WALD00001) |
| `requestSource` | Enum | 필수 | 요청 출처 (POS, KIOSK, APP, AGENT, EXTERNAL_SYSTEM) |
| `datetime` | String | 필수 | 요청 시간 (ISO 8601 형식, ±5분 이내) |

##### 응답 형식
```json
{
  "data": {
    "token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresAt": "2025-12-17T10:30:00Z"
  }
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `token` | String | JWT 액세스 토큰 |
| `expiresAt` | String | 토큰 만료 시간 (ISO 8601 UTC) |

##### 구현 예시

**Java**
```java
import javax.crypto.Mac;
import javax.crypto.spec.SecretKeySpec;
import java.nio.charset.StandardCharsets;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;
import java.util.Base64;

public class TokenIssueExample {
    private static final String API_ENDPOINT = "https://api.example.com/v0/auth/token/issue";
    private static final String PROJECT_API_KEY = "your-project-api-key";

    public static String issueToken(String projectId, String shopCode, String requestSource)
            throws Exception {
        // 1. 요청 본문 생성
        String datetime = LocalDateTime.now().format(DateTimeFormatter.ISO_LOCAL_DATE_TIME);
        String requestBody = String.format(
            "{\"projectId\":\"%s\",\"shopCode\":\"%s\",\"requestSource\":\"%s\",\"datetime\":\"%s\"}",
            projectId, shopCode, requestSource, datetime
        );

        // 2. Signature 생성
        String signature = generateSignature(PROJECT_API_KEY, requestBody);

        // 3. HTTP 요청 전송
        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(API_ENDPOINT))
            .header("Content-Type", "application/json")
            .header("Waldlust-Signature", signature)
            .POST(HttpRequest.BodyPublishers.ofString(requestBody))
            .build();

        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
        return response.body();
    }

    private static String generateSignature(String secret, String payload) throws Exception {
        Mac hmacSha512 = Mac.getInstance("HmacSHA512");
        SecretKeySpec secretKey = new SecretKeySpec(secret.getBytes(StandardCharsets.UTF_8), "HmacSHA512");
        hmacSha512.init(secretKey);
        byte[] hash = hmacSha512.doFinal(payload.getBytes(StandardCharsets.UTF_8));
        return Base64.getEncoder().encodeToString(hash);
    }
}
```

**Python**
```python
import hmac
import hashlib
import base64
import json
import requests
from datetime import datetime

class TokenIssueClient:
    def __init__(self, project_api_key: str):
        self.project_api_key = project_api_key
        self.api_endpoint = "https://api.example.com/v0/auth/token/issue"

    def issue_token(self, project_id: str, shop_code: str, request_source: str) -> dict:
        # 1. 요청 본문 생성
        datetime_str = datetime.now().strftime("%Y-%m-%dT%H:%M:%S")
        request_body = {
            "projectId": project_id,
            "shopCode": shop_code,
            "requestSource": request_source,
            "datetime": datetime_str
        }
        payload = json.dumps(request_body, separators=(',', ':'))

        # 2. Signature 생성
        signature = self._generate_signature(self.project_api_key, payload)

        # 3. HTTP 요청
        headers = {
            "Content-Type": "application/json",
            "Waldlust-Signature": signature
        }

        response = requests.post(self.api_endpoint, headers=headers, data=payload)
        return response.json()

    def _generate_signature(self, secret: str, payload: str) -> str:
        signature = hmac.new(
            secret.encode('utf-8'),
            payload.encode('utf-8'),
            hashlib.sha512
        )
        return base64.b64encode(signature.digest()).decode('utf-8')

# 사용 예시
if __name__ == "__main__":
    client = TokenIssueClient("your-project-api-key")
    response = client.issue_token("project-123", "WALD00001", "POS")
    print(response)
```

**JavaScript/Node.js**
```javascript
const crypto = require('crypto');
const axios = require('axios');

class TokenIssueClient {
    constructor(projectApiKey) {
        this.projectApiKey = projectApiKey;
        this.apiEndpoint = 'https://api.example.com/v0/auth/token/issue';
    }

    async issueToken(projectId, shopCode, requestSource) {
        // 1. 요청 본문 생성
        const datetime = new Date().toISOString().slice(0, 19);
        const requestBody = {
            projectId: projectId,
            shopCode: shopCode,
            requestSource: requestSource,
            datetime: datetime
        };
        const payload = JSON.stringify(requestBody);

        // 2. Signature 생성
        const signature = this.generateSignature(this.projectApiKey, payload);

        // 3. HTTP 요청
        const response = await axios.post(this.apiEndpoint, requestBody, {
            headers: {
                'Content-Type': 'application/json',
                'Waldlust-Signature': signature
            }
        });

        return response.data;
    }

    generateSignature(secret, payload) {
        const hmac = crypto.createHmac('sha512', secret);
        hmac.update(payload);
        return hmac.digest('base64');
    }
}

// 사용 예시
(async () => {
    const client = new TokenIssueClient('your-project-api-key');
    const response = await client.issueToken('project-123', 'WALD00001', 'POS');
    console.log(response);
})();
```

#### 1.2 Project API Key 유효성 검사 (Project API Key Validation)

토큰 발급 전에 Project API Key의 유효성만 확인하고 싶을 때 사용합니다. 이 엔드포인트는 JWT 토큰을 발급하지 않고, API Key가 유효한지만 검증합니다.

##### API 엔드포인트
```
POST /v0/auth/api-key/validate
```

##### 요청 헤더
```http
Content-Type: application/json
Waldlust-Signature: {HMAC-SHA512 서명}
```

##### 요청 본문
```json
{
  "projectId": "프로젝트 ID",
  "version": 1,
  "requestSource": "POS | KIOSK | APP | AGENT | EXTERNAL_SYSTEM",
  "datetime": "2025-12-10T10:30:00"
}
```

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| `projectId` | String | 필수 | 프로젝트 식별자 |
| `version` | Integer | 필수 | API 버전 (현재: 1) |
| `requestSource` | Enum | 필수 | 요청 출처 |
| `datetime` | String | 필수 | 요청 시간 (±5분 이내) |

##### 응답 형식

| 필드 | 타입 | 설명 |

## 토큰 사용 방법

### Bearer Token 형식

발급받은 JWT 토큰은 HTTP `Authorization` 헤더에 Bearer 스키마로 포함하여 사용합니다.

```http
GET /v0/some-api-endpoint
Authorization: Bearer eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...
Content-Type: application/json
```

### 토큰 만료 및 갱신

| 속성 | 값 |
|------|------|
| **토큰 유효 기간** | 7일 |
| **만료 시간 확인** | 응답의 `expiresAt` 필드 참조 |
| **갱신 방법** | 기존 인증 방법으로 새 토큰 재발급 |

⚠️ **중요**: 토큰 갱신(refresh) 엔드포인트는 별도로 제공되지 않습니다. 토큰이 만료되기 전에 동일한 인증 방법을 사용하여 새로운 토큰을 발급받아야 합니다.

### 토큰 갱신 전략 권장사항

```
토큰 발급 시점         토큰 갱신 권장 시점          토큰 만료
      │                      │                      │
      ├──────────────────────┼──────────────────────┤
      0일                   5-6일                  7일

→ 만료 1-2일 전에 새 토큰 발급 권장
```

## Signature 생성 가이드

Project API Key 인증 방법에서 사용하는 Signature는 HMAC-SHA512 알고리즘을 사용합니다.

### Signature 생성 단계

```
1. Payload 준비
   ↓
2. HMAC-SHA512 해시 생성 (Secret Key: Project API Key)
   ↓
3. Base64 인코딩
   ↓
4. Waldlust-Signature 헤더에 포함
```

### Payload 생성 규칙

⚠️ **중요**: Payload는 **JSON 문자열**이며, 공백 없이 압축된 형태여야 합니다.

**올바른 예시**:
```json
{"projectId":"project-123","shopCode":"WALD00001","requestSource":"POS","datetime":"2025-12-10T10:30:00"}
```

**잘못된 예시** (공백 포함):
```json
{
  "projectId": "project-123",
  "shopCode": "WALD00001",
  "requestSource": "POS",
  "datetime": "2025-12-10T10:30:00"
}
```

### Datetime 검증

API 서버는 요청 시간(`datetime` 필드)이 서버 시간 기준 **±5분 이내**인지 검증합니다.

| 서버 시간 | 요청 시간 | 결과 |
|----------|----------|------|
| 10:30:00 | 10:28:00 | ✓ 허용 (2분 차이) |
| 10:30:00 | 10:35:00 | ✓ 허용 (5분 차이) |
| 10:30:00 | 10:36:00 | ✗ 거부 (6분 차이) |
| 10:30:00 | 10:24:00 | ✗ 거부 (6분 차이) |

💡 **권장사항**: 시스템 시간이 NTP(Network Time Protocol) 서버와 동기화되어 있는지 확인하세요.

### 언어별 구현 예시

#### Java
```java
import javax.crypto.Mac;
import javax.crypto.spec.SecretKeySpec;
import java.nio.charset.StandardCharsets;
import java.util.Base64;

public class SignatureGenerator {
    public static String generateSignature(String secret, String payload) throws Exception {
        // 1. HMAC-SHA512 인스턴스 생성
        Mac hmacSha512 = Mac.getInstance("HmacSHA512");

        // 2. Secret Key 설정
        SecretKeySpec secretKey = new SecretKeySpec(
            secret.getBytes(StandardCharsets.UTF_8),
            "HmacSHA512"

        hmacSha512.init(secretKey);

        // 3. HMAC 해시 생성
        byte[] hash = hmacSha512.doFinal(payload.getBytes(StandardCharsets.UTF_8));

        // 4. Base64 인코딩
        return Base64.getEncoder().encodeToString(hash);
    }
```

#### 문제 1: INVALID_SIGNATURE 에러

**증상**:
```json

    "code": "INVALID_SIGNATURE",
    "message": "Signature validation failed"
  }
```

**원인**:

```python
payload = json.dumps(request_body, separators=(',', ':'))
print(f"Payload: {payload}")

# 3. Signature 생성 확인
signature = hmac.new(
    project_api_key.encode('utf-8'),  # UTF-8 인코딩 확인
    payload.encode('utf-8'),
    hashlib.sha512
)
signature_b64 = base64.b64encode(signature.digest()).decode('utf-8')
print(f"Signature: {signature_b64}")
```

```python
# 토큰 만료 시간 확인 및 자동 갱신
import json
from datetime import datetime, timedelta

def get_valid_token():
    # 저장된 토큰 정보 로드
    with open('token.json', 'r') as f:
        token_info = json.load(f)

    expires_at = datetime.fromisoformat(token_info['expiresAt'].replace('Z', '+00:00'))

    # 만료 24시간 전이면 갱신
    if datetime.now() > expires_at - timedelta(days=1):
        print("Token is expiring soon, refreshing...")
        token_info = issue_new_token()

        # 새 토큰 저장
        with open('token.json', 'w') as f:
            json.dump(token_info, f)

    return token_info['token']
```

### 2. Signature 검증의 중요성

- **재전송 공격 방지**: Datetime 검증(±5분)으로 오래된 요청 차단
- **변조 방지**: HMAC-SHA512 서명으로 요청 무결성 보장
- **상수 시간 비교**: 타이밍 공격 방지

### 3. HTTPS 사용 필수

⚠️ **경고**: HTTP를 사용하면 API Key와 토큰이 평문으로 전송되어 중간자 공격(MITM)에 노출됩니다.

✅ **필수 조치**:
```python
# HTTPS 엔드포인트만 사용
API_ENDPOINT = "https://api.example.com"  # ✓

# HTTP는 절대 사용 금지
API_ENDPOINT = "http://api.example.com"  # ✗
```

## 참고사항

### RequestSource 타입

External API는 다양한 클라이언트 유형을 지원합니다. 요청 출처에 따라 적절한 `requestSource` 값을 사용하세요.

| RequestSource | 설명 | 사용 사례 |
|--------------|------|----------|
| `POS` | Point of Sale 시스템 | 매장 POS 단말기 |
| `KIOSK` | 키오스크 단말기 | 무인 주문 키오스크 |
| `APP` | 모바일 애플리케이션 | iOS, Android 앱 |
| `AGENT` | 매장 관리 에이전트 | 백오피스 관리 시스템 |
| `EXTERNAL_SYSTEM` | 외부 시스템 | 파트너사 시스템, 3rd party 통합 |


### API Rate Limiting

현재 API Rate Limiting 정책은 별도로 공지되지 않았습니다. 과도한 요청으로 인한 서비스 거부를 방지하기 위해 다음 권장사항을 따르세요:

**권장사항**:
- 토큰 발급: 필요할 때만 발급 (캐시 활용)
- API 호출: 불필요한 중복 요청 방지
- 에러 재시도: Exponential backoff 전략 사용
